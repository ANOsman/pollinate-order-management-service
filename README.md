# Order Management Microservice

A Spring Boot microservice for managing **Products** and **Orders**.

## Tech Stack

* Java 21
* Spring Boot
* Spring Data JPA
* Spring Security (Basic Auth)
* H2 In-Memory Database
* Project Lombok
* Swagger / OpenAPI
* JUnit + Mockito

---

## Architecture

The application follows a **layered architecture**:

Controller -> Service -> Repository -> Database

* **Controller** – REST API endpoints
* **Service** – business logic
* **Repository** – data access via Spring Data JPA
* **Entity / DTO** – persistence and API models

---

## API Endpoints


### Products

```
POST /products
GET /products/{id}
GET /products
```

### Orders

```
POST /orders
GET /orders/{id}
GET /orders
```

---

## Business Rules

When creating an order:

* All provided **product IDs must exist**
* The **total price is calculated automatically**
* If any product does not exist, the order request is rejected

---

## Database Schema
```
Since an Order can contain many Products and a Product can appear in many orders,
Orders and products have a **many-to-many relationship**.
```

---

Tables:

* `products`
* `orders`
* `order_products` (join table)

The **total price is stored on the order** to avoid recalculating totals.

The schema is automatically generated by **Hibernate using JPA entities**.
